GitHub Sync

Pulls data from the GitHub organisation into the knowledge graph. Start with Members and Repos, then run Stats since contributions depend on the people and projects from those two steps. Engagements should run after Stats, and Worked For derives the person-to-client connections. Lifecycle closes out engagements that have gone quiet.

Members
Creates a cc:Person for each org member with their name, GitHub profile and avatar.
Started Error
Repos
Creates a cc:Project for each repo with its name, description, URL, languages and topics.
Started Error
Stats
Updates contribution counts per person per project: commits, PR reviews, merged PRs and lines added.
Started Error
Engagements
Creates one cc:Engagement per client org, dated from their earliest project or contribution. Slack sync and Stats need to have run at least once first.
Started Error
Worked For
Adds cc:workedFor links connecting each person to the client organisations they contributed to through GitHub project work.
Started Error
Lifecycle
Marks engagements as ENG_COMPLETED when there have been no commits to a client's projects in over 12 months.
Started Error
Full Sync
Runs the complete pipeline in dependency order: seed organisations → Slack → GitHub (members, repos, stats, engagements, worked-for, lifecycle). Use this for a proper end-to-end sync — engagements need the Slack-sourced organisation, so GitHub alone is not enough.

Slack Sync

Pulls Slack workspace data into the knowledge graph. Each Slack user is matched to an existing cc:Person by email first, then by name, and the matched person gets their Slack ID and email filled in. Slack User Groups are imported as cc:Team instances. Needs app.slack.token configured in application.yml.

Sync Slack
Matches Slack users to cc:Person records and imports User Groups as cc:Team instances.
Started Error — check token

All operations run in the background. Check backend logs for match and create counts per person.

Seed Data

Loads known organisation records from the curated seed file (seed/known-organizations.json). Safe to run multiple times since it uses upsert semantics and will not overwrite anything you have set manually.

Organizations
Seeds 9 known client organisations (SCL, FOITT, ETF, ERA, EC, Cedefop, BMW, European Parliament) with legal name, website, type, status and founding date. ETF covers both its own and ACQF projects; EC covers ESCO, CORDIS, ELM and Qualification projects.
